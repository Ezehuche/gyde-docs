Connecting Gyde to Klaviyo creates or updates a Klaviyo profile every time a shopper completes a quiz, and fires a **Quiz Completed** event with the shopper's answers as event properties. Use this to trigger personalised post-quiz email flows and SMS sequences.

## Requirements

Your quiz must include an **email question** — Klaviyo profiles require an email address. Without one, the integration will not create profiles.

## Set up the integration

1. Open the quiz in the Gyde app.
2. Go to **Quiz → Integrations**.
3. Click **Connect** next to **Klaviyo**.
4. Fill in the settings and click **Connect with Klaviyo**.

## Settings

### Private API Key (required)

Your Klaviyo **Private API Key** (not the Public API Key). Find it in your Klaviyo account under **Account → Settings → API Keys**.

### List ID (optional)

The ID of a Klaviyo list to subscribe new contacts to. Leave blank if you only want to create profiles and fire events without list subscription.

Find the List ID in Klaviyo under **Lists & Segments → your list → Settings**.

### Field mappings (optional)

Map specific quiz questions to named Klaviyo profile properties. For example, map a "Skin type" question to a `skin_type` Klaviyo property so you can use it in flow filters and segmentation.

If you leave field mappings blank, Gyde will auto-generate property names from each question title.

## What gets sent to Klaviyo

On each quiz submission, Gyde:

1. **Upserts a profile** — creates a new profile if the email doesn't exist, or updates it if it does. Quiz answers are stored as profile properties.
2. **Tracks a "Quiz Completed" event** — fires an event with the quiz name, handle, answers, and recommended products as event properties.
3. **Subscribes to list** (if List ID is configured) — subscribes the contact to the list with email marketing consent.

## Use cases

- **Post-quiz welcome flow** — trigger a Klaviyo flow on the "Quiz Completed" event and send a personalised product recommendation email.
- **Segmented campaigns** — filter on profile properties (e.g. `skin_type = "oily"`) to send targeted promotions.
- **Browse abandonment** — combine quiz profile data with Klaviyo's browse/cart abandonment flows for richer personalisation.
