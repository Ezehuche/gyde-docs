Connecting Gyde to Omnisend creates or updates an Omnisend contact whenever a shopper completes a quiz, and fires a **Quiz Completed** event with quiz answers as custom properties. Use this to trigger post-quiz email and SMS automations.

## Requirements

Your quiz must include an **email question** — Omnisend contacts require an email address. Without one, the integration will not sync contacts.

## Set up the integration

1. Open the quiz in the Gyde app.
2. Go to **Quiz → Integrations**.
3. Click **Connect** next to **Omnisend**.
4. Fill in the settings and click **Connect with Omnisend**.

## Settings

### API Key (required)

Your Omnisend **API Key**. Find it in your Omnisend account under **Store Settings → Integrations & API → API Keys**.

### List ID (optional)

The ID of an Omnisend list to subscribe new contacts to. Leave blank to create contacts without list assignment.

### Field mappings (optional)

Map specific quiz questions to named Omnisend custom properties. For example, map a "Hair type" question to a `hair_type` property so you can filter on it in automation triggers.

If left blank, Gyde auto-generates property names from each question title.

## What gets sent to Omnisend

On each quiz submission, Gyde:

1. **Upserts a contact** — creates a new contact if the email doesn't exist, or updates it. Quiz answers are stored as custom properties on the contact.
2. **Tracks a "Quiz Completed" event** — fires an event with the quiz name, answers, and recommended products. Use this as an automation trigger in Omnisend.
3. **Subscribes to list** (if List ID is configured).

## Use cases

- **Post-quiz automation** — trigger an Omnisend automation on the "Quiz Completed" event and send a personalised product recommendation email.
- **Segmentation** — filter contacts by quiz answer properties to create targeted segments for campaigns.
- **Cross-sell sequences** — build a follow-up sequence based on which products were recommended in the quiz.
