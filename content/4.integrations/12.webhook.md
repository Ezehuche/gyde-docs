Webhooks let you receive quiz submissions as they happen — in real time — at any endpoint you control. Use them to pipe quiz data into your own systems, trigger automation, or build custom post-quiz flows.

## How it works

Whenever a shopper submits a quiz, Gyde sends a **POST** request to your endpoint URL with the full submission data as a **JSON** payload.

## Add a webhook

1. Open the quiz in the Gyde app.
2. Go to **Quiz → Integrations**.
3. Click **Connect** next to **Webhook**.
4. Enter your endpoint URL and click **Save**.

## Endpoint requirements

Your endpoint must:

- Accept **POST** requests with a `Content-Type: application/json` body.
- Return a **2XX** status code within **10 seconds** (or Gyde will retry).

```json
  "formId": "TEjgpIyO",

      "value": {

      "id": "q2def",

  "submittedAt": "2026-03-14T10:30:00.000Z"
}
```

## Common use cases

- **Klaviyo / email marketing** — send a quiz-results email containing the recommended products.
- **CRM** — create or update a customer record with their quiz profile.
- **Custom analytics** — log quiz data alongside your own store analytics.
- **Loyalty / rewards** — award points when a shopper completes a quiz.

For no-code automation, see the [Zapier](/integrations/zapier) integration.
