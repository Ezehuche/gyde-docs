Conditional logic lets you build branching quiz flows — showing or hiding questions, or jumping straight to a specific slide — based on how a shopper answers. This keeps the quiz relevant and concise for every shopper, regardless of their profile.

## Why use conditional logic in a recommendation quiz?

- **Personalise the path**: A shopper who selects "Oily skin" should see different follow-up questions than one who selects "Dry skin".
- **Skip irrelevant questions**: If a shopper says they don't use supplements, skip the supplement-frequency question entirely.
- **Route to a result early**: If a shopper's answers clearly match one product, send them to the Thank You / results slide without asking further questions.
- **Segment by intent**: Show beginner-focused questions to shoppers who self-identify as new to the category, and expert-focused questions to experienced shoppers.

## How to create a logic rule

1. Open the quiz builder and click **Logic** in the top toolbar.
2. Click the question you want to add logic to.
3. In the modal that opens, click **Add rule**.
4. Configure the rule:
   - **If** — select the question and the answer(s) that trigger the rule.
   - **Then** — choose the action: `Jump to`, `Show`, or `Hide` a specific question.
5. Click **Save**.

You can add multiple rules to a single question. Rules are evaluated in order — the first matching rule wins.

## Rule actions

| Action | Behaviour |
|---|---|
| **Jump to** | Skip forward (or backward) to a specific question slide |
| **Show** | Show a question that is hidden by default |
| **Hide** | Hide a question that is shown by default |

## Example: Skin type quiz

A skincare quiz might have these rules on the "What is your skin type?" question:

- If answer is **Oily** → Jump to "What is your main skin concern for oily skin?"
- If answer is **Dry** → Jump to "What is your main skin concern for dry skin?"
- If answer is **Combination** → Show the "Which zone bothers you most?" question (hidden by default)

This creates three distinct quiz paths from a single branching point, each mapping to different product choices.

## Combining logic with product mapping

Conditional logic and product mapping work together. A shopper routed to a specific branch will only encounter the choice options on that branch — meaning their product recommendations are scoped to the products mapped to those specific choices.

This is the most powerful way to build a precise recommendation engine: segment by profile first (via logic), then map curated products to each segment's choices.
